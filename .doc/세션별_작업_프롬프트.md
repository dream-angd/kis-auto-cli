# KIS 자동매매 CLI 개선 작업 — forge 프롬프트

> 입력 소스: `.doc/개선_할_일_목록.txt` (총 14개 개선 항목)
> 작성일: 2026-05-02
> forge command가 **작업 분할·계획·설계·구현·검증까지 일임**한다. 사용자는 세션마다 아래 프롬프트만 붙여넣으면 된다.

---

## 운용 방식

- forge가 직접 `.doc/개선_할_일_목록.txt`를 읽고, 항목 간 의존성·작업 규모를 분석해 **세션별 작업 단위를 스스로 결정**한다.
- 매 세션은 "이번에 처리할 범위"를 forge가 먼저 제안 → 사용자 확인 → 진행하는 흐름.
- 세션 종료 시 forge가 다음 세션에 인계할 상태(완료 항목, 잔여 항목, 주의사항)를 요약한다.
- 사용자는 **세션 1 프롬프트만 새 대화에서 사용**하면 되고, 이후 세션은 forge가 마지막에 알려준 "다음 세션 프롬프트"를 그대로 사용한다.

---

## 📌 세션 1 (최초 진입) — 전체 분석 + 첫 묶음 처리

새 Claude 세션에서 아래 프롬프트를 그대로 붙여넣는다.

```
/forge

C:\Users\lgw1156\Desktop\lgw\999_개인\kis-auto-cli\.doc\개선_할_일_목록.txt 파일을 읽고 다음 작업을 수행한다.

[1단계] 전체 분석 및 세션 분할 계획 수립
- 14개 개선 항목 전체를 검토한다.
- 각 항목의 영향 범위, 우선순위, 항목 간 의존 관계, 코드 변경 규모를 분석한다.
- 한 세션에서 무리 없이 처리 가능한 단위로 묶어 N개 세션으로 분할한다 (단일 컨텍스트 폭주 방지).
- 분할 결과를 표 형태로 사용자에게 제시한다.
  (세션 번호 / 묶음 주제 / 포함 항목 / 예상 규모 / 권장 순서)
- 분할 계획을 .doc/작업_플랜.md 파일로 저장한다.

[2단계] 사용자 확인
- 분할 계획에 대해 사용자 확인을 받는다.
- 수정 요청이 있으면 반영한다.

[3단계] 첫 번째 세션 작업 수행
- 승인된 첫 세션 범위에 한해 설계 → 구현 → 검증을 진행한다.
- 설계 단계에서 변경할 파일/함수/접근 방식을 먼저 제시하고 사용자 확인을 받는다.
- 구현 후 자체 검증(스모크 테스트, grep 확인 등)을 수행한다.
- 작업 결과 요약(변경 파일, 검증 결과)을 출력한다.

[4단계] 다음 세션 인계 정보 작성
- 완료 항목 / 잔여 항목 / 다음 세션이 알아야 할 주의사항을 정리한다.
- 다음 세션에서 새 대화에 그대로 붙여넣을 "다음 세션 프롬프트"를 .doc/다음_세션_프롬프트.md 파일로 저장한다.
- 이 프롬프트에는 작업_플랜.md 경로, 진행 상태, 이번에 처리할 세션 번호가 명시되어야 한다.

제약사항:
- 입력 파일(.doc/개선_할_일_목록.txt)에 명시된 14개 항목 외 작업은 임의로 추가하지 않는다.
- 각 단계에서 사용자 확인이 필요한 경우 진행을 멈추고 묻는다.
- 한국어로 응답한다.
```

## 📌 세션 2 이후 — 후속 세션

세션 1 종료 시 forge가 `.doc/다음_세션_프롬프트.md`에 자동 생성해둔 프롬프트를 그대로 사용한다.

기본 형태(forge가 채워 넣음):

```
/forge

C:\Users\lgw1156\Desktop\lgw\999_개인\kis-auto-cli\.doc\작업_플랜.md 와
C:\Users\lgw1156\Desktop\lgw\999_개인\kis-auto-cli\.doc\개선_할_일_목록.txt 를 읽고
세션 [N]의 작업 범위만 진행한다.

[수행 단계]
1. 작업_플랜.md에서 이번 세션의 범위와 이전 세션의 인계사항을 확인한다.
2. 세션 범위에 대해 설계안을 제시하고 사용자 확인을 받는다.
3. 구현 → 자체 검증을 수행한다.
4. 결과 요약 + 다음 세션 인계 정보를 .doc/다음_세션_프롬프트.md 에 갱신한다.
   (마지막 세션이면 .doc/작업_완료_보고서.md 를 작성한다.)

제약사항:
- 이번 세션 범위 외 항목은 손대지 않는다.
- 각 단계에서 사용자 확인이 필요한 경우 진행을 멈추고 묻는다.
- 한국어로 응답한다.
```

---

## 사용 가이드

1. **세션 1**: 위 "세션 1 프롬프트" 블록을 새 Claude 세션에 붙여넣기 → forge가 분할 계획 제시 + 첫 묶음 작업
2. **세션 2~N**: forge가 `.doc/다음_세션_프롬프트.md`에 남긴 프롬프트를 새 Claude 세션에 붙여넣기
3. 모든 세션 종료 시 `.doc/작업_완료_보고서.md`로 마무리
4. 각 세션 사이에는 사용자가 직접 `git diff` 검토 → 커밋 권장
