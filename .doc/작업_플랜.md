## 분할 원칙

1. **우선순위(Critical → Low) 우선**, 동일 우선순위 내에서는 의존 관계로 묶음.
2. **같은 파일/같은 상태 변수**를 다루는 항목은 한 세션에 묶어 충돌 방지.
3. **기반 작업(보안, 환경변수 중앙화)을 먼저** 처리하여 후속 세션에서 재작업 최소화.
4. **테스트(11번)는 모든 코드 변경 후** 별도 세션으로 분리하여 안정된 코드 기준으로 작성.
5. 한 세션은 평균 2~3 파일, 50~150 LOC 변경 규모로 제한.

## 의존 관계 메모

- 세션 1의 `src/config.py`가 도입되면 세션 2~5에서 `os.getenv` 직접 호출을 새 통일 API로 호출해야 한다 → 후속 세션은 반드시 세션 1 완료를 전제로 진행.
- 세션 3의 실현손익 계산(`③`)은 세션 2의 `sell()` 결과 구조와 결합될 수 있어 세션 2 후 진행이 안전.
- 세션 4의 `⑧ analyze 보유종목 연동`은 `trader.get_holdings()`를 사용 → 세션 2의 trader 시그니처 변경이 끝난 뒤가 안전.
- 세션 6의 단위 테스트는 모든 모듈 변경이 끝난 후 작성해야 테스트 재작성 비용 최소화.
