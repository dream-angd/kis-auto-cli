## 2. 세션 3에서 주의해야 할 인계사항

1. **세션 1·2의 API 사용 규칙 유지**
   - 환경변수는 `os.getenv()` 직접 호출 금지 → `from src import config; config.get_*()` 사용
   - 새 환경변수 필요 시 `src/config.py`에 함수 추가 후 사용

2. **`sell()` 반환 구조 참고**
   - `src/trader.py`의 `sell()`은 `_order_request` 반환값을 그대로 반환 → KIS API 응답 구조임
   - `③ daily_loss 실현손익 계산`은 매도 체결 후 실현손익을 계산해야 함
   - 현재 `scheduler.py`의 `daily_loss`는 `h["profit_loss"]`(평가손익)을 사용 → 이를 실현손익 누적으로 교체

3. **상태 파일 경로 일관성**
   - `token_cache.json`이 `~/.kis/`에 있으므로 상태 파일도 `~/.kis/state.json`으로 통일 권장
   - `config.py`에 `get_state_path()` 함수 추가하는 방식으로 처리

4. **세션 3 작업이 세션 4에 미치는 영향**
   - `sell()` 반환값에서 실현손익을 추출하는 로직이 추가되면, 세션 4의 `analyze 보유종목 연동(⑧)`도 같은 sell 반환값 구조를 참조할 수 있음 → 인터페이스를 명확히 정의
